# Setting Up Your Kubernetes Control Plane (Master Node)


## Prerequisites



### Step 2: Preparing The Master Node

```bash
ssh ubuntu@k8sm1
# Refresh package information
sudo apt-get update

# Apply available upgrades
sudo apt-get upgrade -y

# Install prerequisite packages for adding repositories securely
sudo apt-get install -y apt-transport-https ca-certificates curl gnupg nfs-common
# Install containerd
sudo apt-get install -y containerd

# Create the necessary configuration directory for containerd
sudo mkdir -p /etc/containerd

# Generate the default containerd configuration file
sudo containerd config default | sudo tee /etc/containerd/config.toml

# Modify the configuration to use the systemd cgroup driver for better integration with Kubernetes
sudo sed -i 's/SystemdCgroup \= false/SystemdCgroup \= true/g' /etc/containerd/config.toml

# Restart containerd to apply the new configuration and enable it to start on boot
sudo systemctl restart containerd
sudo systemctl enable containerd
```

### Step 3: Configuring System Settings for Kubernetes


```bash
# Disable swap immediately
sudo swapoff -a
# Prevent swap from being re-enabled on reboot by commenting it out in fstab
sudo sed -i '/swap/d' /etc/fstab

# Ensure required kernel modules are loaded on boot
cat <<EOF | sudo tee /etc/modules-load.d/k8s.conf
overlay
br_netfilter
EOF

# Load modules immediately
sudo modprobe overlay
sudo modprobe br_netfilter

# Configure necessary sysctl parameters for Kubernetes networking
cat <<EOF | sudo tee /etc/sysctl.d/k8s.conf
net.bridge.bridge-nf-call-iptables  = 1
net.bridge.bridge-nf-call-ip6tables = 1
net.ipv4.ip_forward                 = 1
EOF

# Apply sysctl parameters without needing a reboot
sudo sysctl --system
```

### Step 4: Installing Kubernetes Core Components

```bash
# Install packages needed to manage repositories
sudo apt-get install -y apt-transport-https ca-certificates curl gpg

# Download the official Kubernetes GPG key for verifying package authenticity
curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.34/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg

# Add the Kubernetes apt repository to the system's sources
echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.34/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list

# Update the package index to include the new Kubernetes repository
sudo apt-get update

# Install the specific versions of kubelet, kubeadm, and kubectl
# Specifying versions ensures a consistent cluster environment.
sudo apt-get install -y kubelet=1.34.1-1.1 kubeadm=1.34.1-1.1 kubectl=1.34.1-1.1

# Mark these packages to be held, preventing automatic updates
sudo apt-mark hold kubelet kubeadm kubectl
```

### Step 5: Initializing the Control Plane


```bash
# Initialize the Kubernetes control plane using kubeadm
# --kubernetes-version ensures consistency
# --pod-network-cidr specifies the IP range for pod networking (required by many CNI plugins)
sudo kubeadm init --kubernetes-version=1.34.1 --pod-network-cidr=10.245.0.0/16
```

```bash
# Create the .kube directory in the user's home directory if it doesn't exist
mkdir -p $HOME/.kube

# Copy the administrative config file generated by kubeadm
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config

# Set the correct ownership so the regular user can read the config file
sudo chown $(id -u):$(id -g) $HOME/.kube/config
```

### Step 7: Verifying the Control Plane Status

**Concept:** Let's perform a quick check to see if our control plane is up and running. We'll use `kubectl` to query the cluster state.

**Implementation:**

```bash
# Check the status of nodes registered with the cluster
# Expect to see the master node listed here.
kubectl get nodes

# Check the status of pods running in the kube-system namespace
# These are the core control plane components.
kubectl get pods -n kube-system
```



```bash
# Remove the taint that prevents scheduling on the control plane node
kubectl taint nodes --all node-role.kubernetes.io/control-plane-
```
